---
title: Setup Self-Hosted GitHub Runner
description: Learn how to set up a self-hosted GitHub Actions runner to use custom hardware, local network resources, or a specific operating system.
---

> With self-hosted runners, you can create custom hardware configurations that meet your needs with processing power or memory to run larger jobs, install software available on your local network, and choose an operating system not offered by GitHub-hosted runners.
> — [GitHub Docs](https://docs.github.com/en/actions/hosting-your-own-runners/managing-self-hosted-runners/about-self-hosted-runners)

<Steps>
  ### Open your repository settings

  Go to your repository on GitHub, then navigate to **Settings > Actions > Runners** and click **New self-hosted runner**.

  ### Choose your Runner Image and Architecture

  Select the operating system and architecture that matches your device. Note that the architecture depends on your device specifications (e.g., `x64`).

  ### Download the runner files

  All required commands are shown on the same page where you selected the Runner Image and Architecture.

  <CodeGroup>
  ```bash title="Linux Runner"
  # Create a folder under drive root
  mkdir actions-runner && cd actions-runner

  # Download the latest runner installer file
  curl -o actions-runner-linux-x64-2.311.0.tar.gz -L https://github.com/actions/runner/releases/download/v2.311.0/actions-runner-linux-x64-2.311.0.tar.gz

  # Extract the installer
  tar xzf ./actions-runner-linux-x64-2.311.0.tar.gz
  ```

  ```powershell title="Windows Runner"
  # Create a folder under the drive root
  mkdir actions-runner; cd actions-runner

  # Download the latest runner installer file
  Invoke-WebRequest -Uri https://github.com/actions/runner/releases/download/v2.311.0/actions-runner-win-x64-2.311.0.zip -OutFile actions-runner-win-x64-2.311.0.zip

  # Extract the installer
  Add-Type -AssemblyName System.IO.Compression.FileSystem ; [System.IO.Compression.ZipFile]::ExtractToDirectory("$PWD/actions-runner-win-x64-2.311.0.zip", "$PWD")
  ```
  </CodeGroup>

  ### Configure and start the runner

  <CodeGroup>
  ```bash title="Linux Runner"
  # Configure the runner
  ./config.sh --url <github-repo-url> --token <token> --labels <lists-of-label>

  # Start the runner
  ./run.sh
  ```

  ```powershell title="Windows Runner"
  # Configure the runner
  ./config.cmd --url <github-repo-url> --token <token> --labels <lists-of-label>

  # Start the runner
  ./run.cmd
  ```
  </CodeGroup>

  <Note>
    Programmatically assigning [labels](https://docs.github.com/en/actions/hosting-your-own-runners/managing-self-hosted-runners/using-labels-with-self-hosted-runners) to a self-hosted runner is optional.
  </Note>

  ### Use your self-hosted runner in a workflow

  Create a new file at `.github/workflows/test.yaml`:

  ```yaml title=".github/workflows/test.yaml"
  name: Test runner
  on:
    push:
  jobs:
    build:
      runs-on: ["self-hosted"]
      steps:
        - name: Checkout repository
          uses: actions/checkout@v4
        - name: List directory
          run: ls
  ```

  ### (Optional) Run as a service

  You can configure the [self-hosted runner application as a service](https://docs.github.com/en/actions/hosting-your-own-runners/managing-self-hosted-runners/configuring-the-self-hosted-runner-application-as-a-service) so it starts automatically on boot.
</Steps>
