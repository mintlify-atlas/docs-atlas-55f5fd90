---
title: Install Docker Engine on WSL
description: Learn how to install Docker Engine on WSL (Windows Subsystem for Linux) step by step.
---

Before installing Docker Engine on WSL, make sure you have WSL installed and running. If not, follow the [Mastering WSL](/blog/mastering-wsl) guide first.

<Steps>
  ### Set up Docker's `apt` repository

  The main reason to set up Docker's `apt` repository is to ensure you're **installing the latest version of Docker Engine** on WSL. By default, the `apt` repository only contains the stable version, which may not be the latest.

  ```bash
  # Add Docker's official GPG key:
  sudo apt update
  sudo apt install ca-certificates curl
  sudo install -m 0755 -d /etc/apt/keyrings
  sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
  sudo chmod a+r /etc/apt/keyrings/docker.asc

  # Add the repository to Apt sources:
  sudo tee /etc/apt/sources.list.d/docker.sources <<EOF
  Types: deb
  URIs: https://download.docker.com/linux/ubuntu
  Suites: $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}")
  Components: stable
  Signed-By: /etc/apt/keyrings/docker.asc
  EOF

  sudo apt update
  ```

  ### Install the Docker packages

  ```bash
  sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
  ```

  By default, the Docker service will start automatically after installation. Verify it with:

  ```bash
  sudo systemctl status docker
  ```

  If it is disabled, enable it by running:

  ```bash
  sudo systemctl enable docker
  sudo systemctl start docker
  ```

  ### Verify the installation

  ```bash
  sudo docker run hello-world
  ```

  ### Add your user to the docker group (Recommended)

  <Note>
    Refer to the [Docker post-install docs](https://docs.docker.com/engine/install/linux-postinstall) for more details.
  </Note>

  By default, you need `sudo` access to run Docker commands. To avoid this, add the current user to the `docker` group.

  ```bash
  sudo groupadd docker
  sudo usermod -aG docker $USER
  newgrp docker

  # Check the access
  docker run hello-world
  ```
</Steps>
